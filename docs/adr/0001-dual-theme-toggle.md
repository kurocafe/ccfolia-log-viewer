# 0001. デザインのデュアルテーマ化（cool / soft）

## ステータス

採用

## 背景

初期のUIは「かっこいい」ダーク基調（クトゥルフ・オカルト調）のみだった。
しかし、もっと「ゆるふわ」な雰囲気（パステル＋手書きフォントの「ろぐびゅ〜あ〜」イメージ）でも使いたいという要望があった。

一方で、既存のデザインを気に入っているケースもあるため、片方を捨てるのではなく、両方を選べるようにしたい。

## 決定

CSSカスタムプロパティ（CSS変数）でテーマを切り替えるデュアルテーマ構成を採用する。

- `<html>` の `data-theme` 属性で `cool`（既定）/ `soft` を切り替える
- 色・フォント・角丸・影などはすべて `:root` と `[data-theme="soft"]` の CSS変数として定義し、コンポーネントは `var(--x)` を参照する
- 選択中のテーマは `localStorage` に永続化する（`useTheme` フック）
- 表示テキストもテーマごとに出し分けるため `themeCopy.ts` に文言マップを用意する
- 切り替え時は「ぬるっと」と滑らかに遷移させる。背景グラデーションは直接補間できないため、ベース色＋不透明度フェードのオーバーレイ（`body::before`）と、`@layer base` の全要素トランジションで実現する

## 結果

- ユーザーがトグルボタン一つで好みのテーマに切り替えられ、設定は再訪時も保持される
- 新しい配色を足したくなった場合も、CSS変数を1か所追加するだけで全体に反映できる
- コンポーネント側は色をハードコードせず `var(--x)` 参照に統一されたため、見た目の変更が局所化された
- トレードオフとして、文言・色の二重管理（`themeCopy.ts` と CSS変数）が発生する。テーマを増やすほど管理コストが上がる点には注意する
