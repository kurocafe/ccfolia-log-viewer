# 0002. 生1d100の判定反映と CC/CCB の推定

## ステータス

採用

## 背景

ccfoliaのログには、`CC` / `CCB` のような技能判定だけでなく、技能値（`<=XX`）を伴わない素の `1D100` だけのロールも含まれる。

これらは「何の技能か」「目標値はいくつか」が判別できないため、これまで集計対象外だった。
しかし、クリティカル／ファンブルの回数としてはカウントしたいという要望があった。

課題は2つ：

1. 技能・目標値が不明なので、成功/失敗の判定はできない
2. クリティカルの範囲が版によって違う（CoC7版=CCは出目1のみ、CoC6版=CCBは出目1〜5）が、生1d100だけでは版を判別できない

## 決定

生1d100を、技能判定とは別系統の `D100Roll` 型（`charName` と `roll` のみ）として扱う。

- 反映するかどうかはチェックボックス（`includeD100`）で切り替え可能にする
- 技能・目標値が不明なため、**成長判定には一切影響させない**
- クリティカル／ファンブルのみをカウントし、それ以外の出目（中間値）は合計回数にのみ加算する
  - ファンブル: 出目 96〜100
  - クリティカル: 版に応じて 1 または 1〜5
- 版の判別は、**同一シナリオ中で CC と CCB が混在しない**という前提のもと、周辺のCC/CCBログの多数決で推定する
  - CCが優勢なら CC シナリオとみなしてクリティカルを出目1のみとする
  - それ以外（参照ログがない場合を含む）は **既定で CCB** とみなしてクリティカルを出目1〜5とする

## 結果

- 技能値なしの素ロールも、クリティカル／ファンブル回数として集計に反映できるようになった
- 成長判定のロジックは従来どおり技能判定（CC/CCB）のみを対象とし、生1d100の影響を受けない
- 版の推定はヒューリスティックであり、シナリオ途中で版が切り替わるような例外的ログでは誤判定の可能性がある。基本方針として既定はCCBとする
- この推定挙動は `analyzer.test.ts` でテスト化されている（CCBは1〜5、CCは1のみ、96〜100はファンブル、中間値は合計のみ）
