# plan

実装計画を立案してから作業を開始します。

## 手順

1. タスクの要件を整理し、曖昧な点を明確にする
2. 影響範囲のファイルを特定する
3. 実装方針を検討し、トレードオフを整理する
4. ステップバイステップの実装計画を作成する
5. ユーザーに計画を提示し、承認を得てから実装を開始する

## 出力形式

```
## 要件の整理
- ...

## 影響ファイル
- path/to/file.ts — 変更内容

## 実装方針
...

## 実装ステップ
1. ...
2. ...
3. ...
```
