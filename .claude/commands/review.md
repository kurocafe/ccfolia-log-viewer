# /review

コードレビューを実施します。

## 手順

1. `git diff` で変更範囲を確認する（引数でファイル・PR の指定があればそれを対象にする）
2. `reviewer` エージェントに差分と対象ファイルを渡してレビューを委譲する
   （`Agent(subagent_type="reviewer", ...)` を使う。bash からは起動できない）
3. エージェントの指摘を確認し、下記の出力形式で報告する

## レビュー観点

- **正確性**: ロジックが要件を満たしているか
- **セキュリティ**: 脆弱性（XSS、SQLインジェクション、認証漏れ等）がないか
- **パフォーマンス**: 不要な処理・N+1・メモリリークがないか
- **可読性**: コードが明快で意図が伝わるか
- **テスト**: 重要なケースがカバーされているか
- **設計**: 責務が適切に分離されているか

## 出力形式

各指摘は重要度（🔴 Critical / 🟡 Warning / 🔵 Suggestion）で分類してください。

```
## レビュー結果

### 🔴 Critical
- file:line — 指摘内容

### 🟡 Warning
- file:line — 指摘内容

### 🔵 Suggestion
- file:line — 指摘内容

## 総評
...
```
