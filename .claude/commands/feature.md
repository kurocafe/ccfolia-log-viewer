# /feature

新機能を実装します。**このコマンドの実行は「実装して」という明示的な指示として扱います**（CLAUDE.md の「実装はユーザー自身が書く」ルールの例外条件を満たす）。

## 使い方

```
/feature ダイスログを CSV でエクスポートできるようにしたい
/feature 検索結果にページネーションを追加
```

## 手順

1. **要件確認**: 機能のゴールと完了の定義を確認する。曖昧な点があれば実装前に質問する（最大3問にまとめて聞く）
2. **既存把握**: 類似機能・再利用できる既存コードを探す
3. **設計**: 影響ファイルと変更方針を短く提示する。大規模（5ファイル超・複数工程）なら /plan への切り替えを提案する
4. **実装**: 既存のパターンと coding-style スキルに従って実装する
5. **テスト**: 主要ケースのテストを書き、実行して通す
6. **動作確認**: 実際に動かして確認する

## 出力形式

```
## 実装した機能
...

## 変更ファイル
- path — 変更内容（1行）

## テスト結果
X件パス（追加: Y件）

## 動作確認
実行したコマンドと確認した挙動

## 未対応・次の候補
...
```

## 完了条件

- [ ] 要件のゴールを満たしている
- [ ] テストが追加され、全テストがパスしている
- [ ] 実際に動かして確認済み（不可能な場合は理由と代替の確認方法を明記）

## 注意点

- 要求されていない機能・抽象化・エラー処理を足さない
- 悪い例: `将来のために設定オプションも用意しておきました` ／ 良い例: `依頼された機能のみ実装。拡張案は「次の候補」に記載`
- 大きすぎる機能は分割を提案する（1回のセッションで完結できる粒度に）
