# /debug

症状から根本原因を特定し、修正します。対症療法ではなく原因を直します。

## 使い方

```
/debug ログイン後にダッシュボードが真っ白になる
/debug TypeError: Cannot read properties of undefined (reading 'map')
```

## 手順

1. **再現**: 症状を再現する最小手順を確立する。再現できない場合は、修正より先に再現条件の調査を優先する
2. **仮説**: 原因候補を確度順に最大3つ挙げる
3. **検証**: 各仮説をログ・デバッガ・最小テストで検証する。**証拠が得られるまで修正しない**
4. **修正**: 根本原因を修正する。症状を隠すだけの修正は禁止
5. **確認**: 再現手順で症状が消えたこと、既存テストが全て通ることを確認する
6. **再発防止**: 同じバグを検出するテストを追加する

## 出力形式

```
## 症状
...

## 根本原因
path/to/file.ts:42 — 何が起きていたか

## 修正内容
...

## 検証結果
- 再現手順: 解消を確認
- テスト: X件パス（追加: Y件）
```

## 完了条件

- [ ] 根本原因がコードの特定箇所（ファイル:行）として示されている
- [ ] 再現手順で症状が消えている
- [ ] 再発防止テストが追加されている（不可能な場合は理由を明記）

## 注意点

- **証拠なしに修正しない**。「たぶんこれ」で直すと別のバグを生む
- 悪い例: `try-catch で囲んでエラーを消す` ／ 良い例: `undefined が渡る呼び出し元を特定して修正する`
- 調査中に別のバグを見つけたら、報告した上で今のバグに集中する（同時に直さない）
