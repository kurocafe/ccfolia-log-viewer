# /orchestrate

大きなタスクを分解・委譲・統合する司令塔です。調査や独立した作業をサブエージェントに任せ、本体のコンテキストを温存します。

## 使うべきか（発動条件）

- **使う**: 3ファイル以上、または複数工程（調査→設計→実装→テスト）にまたがるタスク
- **使わない**: 単一ファイルの修正・質問への回答 → 普通に依頼するか個別コマンド（/debug, /feature 等）を使う

## 使い方

```
/orchestrate 認証機能を NextAuth から Auth.js v5 に移行したい
/orchestrate ログ解析モジュール全体のテストカバレッジを上げたい
```

## 手順

1. **分解**: タスクをサブタスクに分割する。各サブタスクは独立して検証可能で、コンテキスト50%以内で完了する粒度にする
2. **依存整理**: 並列実行できるものと順序が必要なものを分ける
3. **委譲**: 調査・探索は `Explore` エージェント、レビューは `reviewer` エージェントへ。並列可能なものは同時に投げる
4. **統合**: 結果を統合し、サブタスク間に矛盾がないか確認する
5. **検証**: 全体としてゴールを満たすかテスト・動作確認で検証する
6. **報告**: 分担・完了/未完了を報告する

## 出力形式

```
## タスク分解
| # | サブタスク | 担当 | 依存 |
|---|-----------|------|------|
| 1 | ... | Explore | なし |
| 2 | ... | 本体 | 1 |

## 実行結果
- サブタスク1: 完了 — 要点

## 統合検証
テスト・動作確認の結果

## 未完了・持ち越し
（あれば理由と次のアクション）
```

## 完了条件

- [ ] 全サブタスクの完了/未完了が明示されている
- [ ] 統合後にテスト・動作確認を実施している
- [ ] 未完了項目に理由と次のアクションが書かれている

## 注意点

- サブエージェントは必ず `Agent` ツールで起動する（bash からは起動できない）
- 委譲先の結果を鵜呑みにしない。重要な結論は本体で裏取りする
- 悪い例: `grep 1回で済む調査に Explore を起動する` ／ 良い例: `複数ディレクトリの横断調査だけ Explore に投げる`
