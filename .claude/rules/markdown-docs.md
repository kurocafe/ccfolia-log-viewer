# ドキュメント規約

## 基本ルール

- ファイルは1トピックに集中させる
- ドキュメント間のリンクは相対パスを使用する（絶対GitHubのURLは使わない）
- 見出しは階層を飛ばさない（`##` の次は `###`）

## ディレクトリ構成

| ディレクトリ | 用途 |
|------------|------|
| `docs/` | ユーザー向けドキュメント |
| `docs/adr/` | Architecture Decision Records |

## ADR（アーキテクチャ決定記録）

重要な設計判断は `docs/adr/NNNN-title.md` に記録する。

```markdown
# NNNN. タイトル

## ステータス
提案中 / 採用 / 廃止

## 背景
なぜこの決定が必要だったか

## 決定
何を決めたか

## 結果
この決定による影響
```
